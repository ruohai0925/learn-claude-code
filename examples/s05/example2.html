## Agent Design & Building

### The 3 Elements of Any Agent:
1. Capabilities — What can it *do*? (tools, APIs, file access)
2. Knowledge — What does it *know*? (system prompt, skills, RAG)
3. Context — What has *happened*? (conversation history, state)

### Design Thinking, Not Code-First
- Start with the user problem, not the implementation
- Identify the minimum set of tools needed
- Write clear system prompts that guide behavior

### Anti-Patterns I'll Help You Avoid
- Over-engineering tool schemas
- Putting too much in the system prompt
- Not giving the model enough autonomy

What's your agent idea? Tell me about the domain, the users, and what you want the agent to accomplish.
